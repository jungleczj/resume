# 首页上传界面设计规范

## 页面目标
- 用户无需登录即可上传简历
- 简洁直观的上传体验
- 支持中英文切换

## 布局结构

```
┌─────────────────────────────────────────────────────────┐
│                    顶部导航栏                            │
│  Logo [CareerFlow]           [语言切换] [登录]          │
├─────────────────────────────────────────────────────────┤
│                                                          │
│                    Hero Section                          │
│                                                          │
│         AI 驱动的简历优化工具                            │
│         让你的简历脱颖而出                               │
│                                                          │
│    ┌──────────────────────────────────────┐            │
│    │                                       │            │
│    │   📄 拖拽或点击上传简历               │            │
│    │   支持 PDF, DOCX (最大 10MB)         │            │
│    │                                       │            │
│    └──────────────────────────────────────┘            │
│                                                          │
│              或                                          │
│                                                          │
│         [🔗 连接 Notion 导入简历]                       │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

## 组件规范

### 1. 上传区域
- 尺寸: 600px × 300px (桌面), 全宽 (移动端)
- 边框: 虚线边框 2px, 圆角 12px
- 拖拽状态: 高亮边框 + 背景色变化
- 图标: 文档图标 48px

### 2. Notion 按钮
- 样式: 次要按钮
- 图标: Notion logo
- 提示: 需要登录

### 3. 语言切换
- 位置: 右上角
- 选项: 中文 / English
- 默认: 根据 geo 判断

## 交互流程

1. 用户访问首页
2. 选择上传方式:
   - 方式1: 上传文件 → 直接进入工作台
   - 方式2: 连接 Notion → 登录 → 授权 → 进入工作台

## 状态处理

- 上传中: 显示进度条
- 上传失败: 错误提示 + 重试按钮
- 文件格式错误: 提示支持的格式
- 文件过大: 提示大小限制
