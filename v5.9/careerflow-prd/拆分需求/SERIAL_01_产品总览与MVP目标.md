# **0. 产品全景总览**
## **0.1 背景与核心问题**

|**用户真实痛点**|**现有方案局限**|
| :- | :- |
|不知道怎么写才能脱颖而出|模板工具只管形式，不解决内容|
|写不出量化成果，数字从哪来|AI写简历生成假数字，HR一眼识穿|
|每次投不同公司要重写，耗时耗力|没有积累沉淀，每次从零开始|
|有经验但不善表达，自我评价过低|没有工具帮助挖掘整理真实成就|

## **0.2 产品核心逻辑**
先帮用户把「已有的事实」整理成「经三档AI美化的成就条目」（成就库），再在投递时按 JD 要求「智能组装」成定制简历。F1/F2 两条路径的成就均经过同一套三档美化标准，入库质量一致。

## T-S01 · 任务清单

### T-S01-1　项目工程初始化
| 项 | 内容 |
|---|---|
| **输入** | 技术选型决策（Next.js 14 App Router / Supabase / Vercel / Upstash Redis / TypeScript） |
| **输出** | 可运行的 monorepo 骨架；`.env.example` 含所有环境变量 key；CI/CD 流水线（GitHub Actions → Vercel Preview） |
| **关键步骤** | `create-next-app` → 配置 TS strict → 集成 ESLint + Prettier + Husky → 配置 Tailwind + shadcn/ui → 接入 Supabase client（browser/server/middleware 三份） → 配置 Upstash Redis client → 配置 `next-intl`（`/zh` / `/en` 路由） |
| **验收标准** | ① `pnpm dev` 正常启动，访问 `/zh` 和 `/en` 均返回 200；② Supabase 连接测试通过（查询 `profiles` 表不报错）；③ Redis ping 返回 PONG；④ Vercel Preview 部署成功；⑤ PR 触发 lint + type-check 全通过 |

### T-S01-2　Vercel Edge Middleware（Geo → 语言路由）
| 项 | 内容 |
|---|---|
| **输入** | `request.geo.country`（Vercel 注入）；用户 Cookie `ui_lang_override`（手动覆盖时写入） |
| **输出** | 重定向到 `/zh/...` 或 `/en/...`；在 Response header 写入 `x-geo-country` 供后续日志使用 |
| **关键步骤** | `middleware.ts` 读取 geo.country → 映射规则（`CN` → `/zh`，其他 → `/en`）→ 若 Cookie 有手动覆盖则优先 Cookie → 执行 `NextResponse.redirect` |
| **验收标准** | ① 模拟 `CN` geo 访问 `/`，自动跳转 `/zh`；② 模拟 `US` geo，跳转 `/en`；③ 手动设置 Cookie `ui_lang_override=en` 后，CN geo 用户也跳转 `/en`；④ `x-geo-country` header 在所有响应中存在 |

### T-S01-3　匿名用户 Cookie 系统
| 项 | 内容 |
|---|---|
| **输入** | 首次访问请求（无任何用户标识） |
| **输出** | httpOnly Cookie `anonymous_id`（UUID v4，7天，SameSite=Lax）；写入 `profiles` 临时记录（可选，延迟写） |
| **关键步骤** | Middleware 检测 Cookie 是否存在 → 不存在则 `crypto.randomUUID()` → `Set-Cookie` 写入响应；有效期内每次请求滚动续期 |
| **验收标准** | ① 全新浏览器访问，Network 响应头中出现 `Set-Cookie: anonymous_id=...`；② DevTools 确认 Cookie 属性：HttpOnly=true、SameSite=Lax、Max-Age=604800；③ 同一浏览器第二次访问，Cookie 值不变（不重复生成）；④ 匿名用户可访问所有功能页面（不被拦截） |

### T-S01-4　i18n 翻译文件基础结构
| 项 | 内容 |
|---|---|
| **输入** | 产品文案设计稿（中英文） |
| **输出** | `messages/zh.json` + `messages/en.json`；覆盖全部静态 UI 文案 |
| **关键步骤** | 按页面模块组织 key（`upload.*`、`workspace.*`、`payment.*`、`auth.*`）→ 配置 `next-intl` provider → 所有硬编码文案替换为 `t('key')` |
| **验收标准** | ① 切换语言路由，所有页面文案完整切换，无 key missing 警告；② `en.json` 与 `zh.json` key 数量一致（CI 检查脚本）；③ 不存在硬编码中文字符串（grep 扫描） |

## AC-S01 · 模块整体验收标准

1. 所有页面在 `/zh` 和 `/en` 路由下均可正常访问
2. CN geo 用户默认落地 `/zh`，非 CN 默认落地 `/en`，手动 Cookie 覆盖有效

5. Vercel Preview 环境与 Production 环境使用独立的 Supabase + Redis 实例
