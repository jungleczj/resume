## **6.2 语言切换器（v5.9 移入工作台）**
语言切换从「上传前设置」移到「工作台工具栏」，随时可改，不影响主流程进入速度。

|**市场**|**工具栏显示**|**切换效果**|
| :- | :- | :- |
|CN市场（cn\_free）|下拉：「中文」「中英双语」「仅英文」；默认「中文」|实时切换简历预览语言；照片区随「仅英文」切换时可手动关|
|EN市场（en\_paid）|固定「English」（无下拉）|无需切换；始终英文|

## **6.3 照片功能（v5.9 极简化）**
v5.9 完全简化：去掉 target\_country / country\_photo\_rules / ATS警告等全部复杂逻辑。改为工具栏「📷照片开关」，用户自主决定要不要加照片。

|**规格项**|**说明**|
| :- | :- |
|显示控制|工具栏「📷照片开关」：开→右侧简历预览显示照片区；关→隐藏（照片文件保留）|
|默认状态|CN市场：默认开；EN市场：默认关（因多数英语系国家不需要，用户可手动开）|
|上传入口|① 工作台照片区点击/拖入（主入口）② F1解析时检测到人像图 → 弹提示「检测到照片，是否使用？」|
|裁剪|内联Crop组件；锁定2:3比例；不支持切换比例（MVP）|
|文件规格|JPG/PNG ≤2MB；存裁剪后版本|
|存储|路径：photos/{userId or anonymousId}/{uuid}/photo.jpg；Storage RLS；匿名48h TTL；登录用户永久|
|导出|开关开 + 有照片 → 服务端base64嵌入PDF/DOCX；开关关 → 不嵌入（即使已上传）|
|模板扩展|MVP 1套模板（照片固定右上角，2:3）；Phase 2A 多模板，通过 photoSlot 配置实现照片动态定位|

## **6.6 成就联动高亮（v5.6）**
- 右侧简历选中某条成就 → 左下成就区自动滚动高亮对应条目
- 左下点击某条成就 → 右侧简历滚动到并高亮对应位置
- 右侧成就被编辑器修改（与成就库版本不同）→ 左下对应条目「已修改」角标
- 点击角标 → 「同步到成就库」/「仅保留在本版本」

## **6.7 版本历史（v5.6）**
- 点击顶部「版本历史」→ 右侧弹出抽屉面板（不遮挡主内容）
- 版本列表：时间戳 + 版本名称（自动命名或重命名）+ 对应JD摘要
- 点击版本 → 右侧简历切换为只读对比预览 → 「恢复到该版本」才生效
- 版本快照触发：主动「保存版本」/ 每次「生成定制简历」/ 导出前自动

## **6.8 编辑器自动保存（三层）**

|**层级**|**存储**|**触发**|**恢复场景**|
| :- | :- | :- | :- |
|Layer 1：实时|TipTap内存|每次编辑|无需|
|Layer 2：本地|localStorage（500ms防抖）|500ms防抖写入|浏览器崩溃恢复|
|Layer 3：服务端|Supabase resume\_versions|操作暂停3s/主动保存/导出前|换设备/清缓存|

- 崩溃恢复：页面重载时比较 localStorage draft vs 服务端版本；draft 更新则提示「检测到未保存的编辑，是否恢复？」
- 简历模板：MVP 1套默认模板（中英双语兼容）；Phase 2A 增加多模板选择

## T-B1 · 任务清单

### T-B1-1　工作台整体布局与可拖拽分隔线
| 项 | 内容 |
|---|---|
| **输入** | 用户访问 `/workspace`（有 `anonymous_id` 或登录态） |
| **输出** | 三区布局（顶部工具栏 + 左侧上下分区 + 右侧编辑器）；分隔线可拖拽调整比例 |
| **关键步骤** | CSS Grid / Flex 布局 → 顶部固定工具栏 → 左右 `ResizablePanelGroup`（`react-resizable-panels`）→ 左侧 `ResizablePanel`（上：JD 区默认 40%，下：成就区默认 60%，中间可拖拽分隔线）→ 右侧编辑器区 → 工作台初始化：并行 fetch `work_experiences` + `achievements` + 最新 `resume_versions` |
| **验收标准** | ① 桌面端（>1024px）左右分栏正常显示，拖拽分隔线左右宽度可调，不超出边界；② 左侧上下分隔线可拖拽，JD 区高度 20%~80% 之间任意调节；③ 移动端（<1024px）自动切换 Tab 布局（简历 Tab / 成就 Tab）；④ 工作台初始加载时间（FCP）≤ 2s（Lighthouse 测量） |

### T-B1-2　TipTap 编辑器初始化与简历模板
| 项 | 内容 |
|---|---|
| **输入** | `resume_versions` 最新快照（TipTap JSON）；`achievements` 数据；`resume_lang` 设置 |
| **输出** | 右侧编辑器渲染简历内容；成就节点带 `data-achievement-id` 属性；占位符高亮橙色 |
| **关键步骤** | 安装 `@tiptap/react` + 自定义扩展：`AchievementNode`（`NodeView`，带 `data-achievement-id`，支持拖拽 drop 目标）、`PlaceholderMark`（橙色高亮 Mark，点击弹内联输入框）→ 加载最新 `resume_versions.content_json` → 若无历史版本，使用 AI 匹配结果生成初始内容 → 编辑器字体/行距与导出 PDF 模板完全对齐（使用相同 CSS 变量） |
| **验收标准** | ① tier=2 成就中的占位符显示为橙色高亮，其他文本正常显示；② 点击橙色占位符，弹出内联输入框，输入内容后占位符被替换；③ 编辑器内容与 PDF 导出排版一致（目视验收：字体/行距/边距相同）；④ 简历内容在编辑器中可自由编辑（增删改）；⑤ `data-achievement-id` 属性在对应成就节点上存在（DevTools 验证） |

### T-B1-3　三层自动保存
| 项 | 内容 |
|---|---|
| **输入** | TipTap `editor.on('update')` 事件；用户点击「保存」；导出触发 |
| **输出** | Layer2：localStorage 写入（500ms 防抖）；Layer3：`resume_versions` 表新增行（3s 防抖 / 主动保存 / 导出前） |
| **关键步骤** | `useEffect` 监听 editor update → 500ms debounce → 写 `localStorage['draft_resume_{id}']`（TipTap JSON + 时间戳）→ 3s debounce → `POST /api/resume-versions`（`trigger='auto_save'`）→ 页面加载时：比较 localStorage 时间戳 vs 服务端 `created_at` → localStorage 更新 → 顶部黄色 Banner「检测到未保存编辑」[恢复][放弃] |
| **验收标准** | ① 编辑器修改后 500ms，`localStorage` key 存在；② 编辑停止 3s 后，`resume_versions` 表新增一行，`trigger='auto_save'`；③ 强制刷新页面，顶部 Banner 出现「检测到未保存的编辑」；④ 点「恢复」，编辑器内容恢复为 localStorage 版本；⑤ 点「放弃」，localStorage 清除，加载服务端版本 |

### T-B1-4　版本历史抽屉
| 项 | 内容 |
|---|---|
| **输入** | `GET /api/resume-versions`（当前用户所有版本）；用户点击「版本历史」 |
| **输出** | 右侧滑出 300px 抽屉，列出版本列表；点击版本切换只读预览；「恢复到该版本」生效 |
| **关键步骤** | 顶部「版本历史」按钮 → 右侧抽屉 `Sheet` 组件（不遮挡主内容，通过 CSS transform 推移右侧编辑器）→ 版本列表（时间戳 + 名称 + JD 摘要）→ 双击名称进入重命名（inline edit）→ 点击版本 → 右侧编辑器切换为 `editable=false` + 灰色背景「历史版本」标签 → 「恢复到该版本」→ 确认弹窗 → 恢复（自动保存当前为新快照） |
| **验收标准** | ① 每次「生成定制简历」后，版本列表新增一条（`trigger='generated'`）；② 版本历史抽屉打开时，右侧编辑器内容区域可见（抽屉不遮挡）；③ 只读预览状态：编辑器不可输入（鼠标指针为 default）；④ 「恢复到该版本」后，编辑器内容变为选中版本，版本列表顶部出现「当前编辑」新快照 |

|---|---|

| **关键步骤** | `<textarea>` 实时计字（右下角 `N/5000`）→ 超 5000 字截断 + Toast 提示 → 「生成定制简历」按钮 → `POST /api/resume/generate`（携带 `jd_text + resume_lang + userId`）→ 后端：提取 JD 关键词 → `matchAchievements()`（T-S03-6）→ 简历组装 → 返回 TipTap JSON → 编辑器更新内容 → 自动保存版本（`trigger='generated'`） |
| **验收标准** | ① 空 JD 点「生成简历（通用版）」，按 ai_score 排序生成；② 有 JD 点「生成定制简历」，生成内容中成就与 JD 关键词相关性更高（人工判断）；③ 生成中编辑器显示骨架加载，生成后立即渲染新内容；④ 生成耗时 ≤ 10s（P95）；⑤ 生成后版本历史多出一条（`trigger='generated'`） |

## AC-B1 · 模块整体验收标准

1. **工作台完整 E2E**：F1 上传完成进入工作台 → 粘贴 JD → 生成 → 拖拽换成就 → 看版本历史 → 导出，全流程无报错，用时 ≤ 5 分钟
2. **自动保存可靠性**：模拟浏览器强制关闭（Command+Q），重新打开后崩溃恢复 Banner 出现，恢复内容与关闭前一致
3. **拖拽兼容性**：在 Chrome / Safari / Firefox / Edge 最新版均可正常拖拽，视觉反馈一致
4. **移动端降级**：iPhone 15 Safari，工作台 Tab 切换正常，「插入/替换」按钮可点击（44px 触控目标）
5. **编辑器与 PDF 一致性**：随机选取 5 份简历，编辑器预览与导出 PDF 的字体/行距/边距目视一致
